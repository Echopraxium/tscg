## Candidate 2: Switch

### Definition
**Mechanism enabling discrete state transition between operational modes (binary or N-ary), often triggered by threshold crossing or control signal. Switch operates over a finite set of discrete states, not limited to binary ON/OFF.**

### Proposed Classification
- **Category**: Regulatory (with Dynamic aspects)
- **Perspective**: Dual (Territory: physical switches; Map: logical transitions)
- **Polarity**: Neutral (N-ary generalization, though binary is special case)
- **Tensor Formula**: `A⊗D⊗I`

### ASFID Decomposition
```
|Switch⟩ = 0.80|A⟩ + 0.75|D⟩ + 0.70|I⟩ + 0.40|S⟩ + 0.30|F⟩
```

**Justification**:
- **A (Attractor)** = 0.80 - HIGH: Bi-stable states (ON/OFF attractors)
- **D (Dynamics)** = 0.75 - HIGH: Rapid state transition
- **I (Information)** = 0.70 - MEDIUM-HIGH: Control signal carries information
- **S (Structure)** = 0.40 - MEDIUM: Physical/logical structure enables switching
- **F (Flow)** = 0.30 - LOW: Switching itself doesn't involve continuous flow

### Transdisciplinary Validation

| Domain | Example | States | Switching Mechanism | Validation |
|--------|---------|--------|---------------------|------------|
| **Electronics** | Transistor | 2 (Binary) | Voltage-controlled switch | ✅ ON/OFF |
| **Home Appliances** | Washing machine programmer | 6-12 (N-ary) | Rotary/Digital selector | ✅ Wash cycles |
| **Biology** | Ion channel gating | 2-3 | Voltage/ligand-gated | ✅ Open/Closed/Inactivated |
| **Photography** | Camera mode dial | 5-10 (N-ary) | Manual selector | ✅ Auto/Manual/Aperture/Shutter/Scene modes |
| **Computing** | Process scheduler | N (N-ary) | Time-slice switching | ✅ CPU context switches |
| **Transportation** | Car transmission | 5-8 (N-ary) | Gear selector | ✅ P/R/N/D/L/2/1 |
| **Audio** | Guitar pickup selector | 3-5 (N-ary) | Toggle switch | ✅ Neck/Middle/Bridge |
| **HVAC** | Thermostat mode | 4 (N-ary) | Dial/Digital | ✅ Off/Heat/Cool/Auto |
| **Optics** | Filter wheel | 6-12 (N-ary) | Rotary mechanism | ✅ Color filter selection |
| **Neuroscience** | Neural firing modes | 3+ (N-ary) | Membrane potential | ✅ Tonic/Burst/Silent |

**Validation Count**: 10 domains ✅

**N-ary Examples Breakdown**:
- Binary (N=2): 3 examples (Transistor, Simple Ion Channel, Simple toggle)
- Ternary (N=3): 2 examples (Ion channels, Audio pickup)
- Quaternary (N=4): 1 example (HVAC)
- N-ary (N>4): 4 examples (Washing machine, Camera, Transmission, Filter wheel)

### N-ary Nature
**Switch is NOT inherently binary** - it operates over a finite discrete set of states:
- **Binary Switch**: 2 states (ON/OFF, Open/Closed)
- **Ternary Switch**: 3 states (Low/Medium/High)
- **N-ary Switch**: N states (Mode selector with N positions)

**Key Property**: Discrete state set with explicit transition mechanism between states.

**Polarity**: Neutral (N-ary is general; binary is special case where N=2)

### Distinction from Existing GenericConcepts

| GenericConcept | Relationship | Distinction |
|-------------|--------------|-------------|
| **Bifurcation** | Related | Bifurcation = CONTINUOUS parameter → qualitative change; Switch = DISCRETE triggered transition among finite states |
| **Threshold** | Complementary | Threshold = CRITICAL VALUE; Switch = MECHANISM using threshold(s) for state transition |
| **Trigger** | Parent | Switch is SPECIFIC type of trigger (discrete multi-state transitions with explicit mechanism) |
| **Activation** | Overlapping | Activation = general enable/disable (binary); Switch = generalized N-ary state selection |
| **Event** | Related | Switch creates Events at transition moments |
| **Mode** | Complementary | Mode = STATES themselves; Switch = TRANSITION MECHANISM between modes |

**Key Distinction**: Switch emphasizes **discrete state transition mechanism over finite set {S₁, S₂, ..., Sₙ}**, with explicit control/trigger, unlike:
- Continuous Bifurcation
- One-way Trigger  
- Static Mode (which is the state, not the transition)

### Binary vs N-ary Switch

**Binary Switch (N=2)**:
- Classical ON/OFF, True/False, Open/Closed
- Simplest case, ubiquitous in digital systems
- Boolean logic foundation

**N-ary Switch (N>2)**:
- Generalization to multiple discrete states
- Examples: Washing machine programs (N=8), Camera modes (N=6), Guitar pickup selector (N=5)
- Mathematically: Switch operates on finite set S = {s₁, s₂, ..., sₙ} with transition function τ: S → S

**Formal Definition**:
```
Switch = (S, τ, C)
where:
  S = {s₁, s₂, ..., sₙ} - Finite state set
  τ: S × C → S - Transition function
  C - Control signal/trigger set
```

**IMPORTANT**: Binary is **special case** where |S| = 2, NOT the defining property of Switch GenericConcept.


**ColorSynthesis** (Optics):
```

```

```

```

```
|Mode⟩ = 0.80|S⟩ + 0.75|I⟩ + 0.75|A⟩ + 0.40|F⟩ + 0.30|D⟩
```

- **I (Information)** = 0.75 - HIGH: Mode encodes specific operational state

## 🔗 Triad Coherence: Layer ↔ Switch ↔ Mode

### Conceptual Relationships

```
┌─────────────────────────────────────────────────┐
│                    LAYER                        │
│   (Spatial/Organizational Stratification)      │
│                                                 │
│  Layer 3: Application Logic                    │
│  Layer 2: Business Rules     ← MODE 2         │
│  Layer 1: Data Access        ← MODE 1         │
│                                    ↑            │
│                                 SWITCH          │
│                            (Transition between  │
│                             modes in layer)     │
└─────────────────────────────────────────────────┘
```

### Triad Properties

1. **Layer** provides SPATIAL organization
2. **Mode** provides BEHAVIORAL states within layers (or across system)
3. **Switch** provides TRANSITION mechanism between modes

### Example: Camera System

**Layers**:
```
Layer 4: User Interface (Menu, Display)
Layer 3: Shooting Logic (Exposure calculation)
Layer 2: Sensor Control (ISO, Duration)
Layer 1: Hardware (Sensor, Shutter, Aperture)
```

**Modes** (at Layer 3):
```
Mode 1: Manual      (User controls ISO, Aperture, Shutter)
Mode 2: Auto        (Camera controls all)
Mode 3: Aperture-P  (User: Aperture; Camera: Shutter, ISO)
Mode 4: Shutter-P   (User: Shutter; Camera: Aperture, ISO)
Mode 5: Scene       (Pre-programmed combinations)
```

**Switch** (Mode Selector):
```
Rotary dial or menu interface that transitions between Mode 1-5
Transition: Mode_i → Mode_j triggered by user input
```

### Formal Relationship

```
System = ⋃(Layers)
Each Layer_i can have Modes = {M₁, M₂, ..., Mₙ}
Switch: Mᵢ → Mⱼ (state transition within or across layers)
```
