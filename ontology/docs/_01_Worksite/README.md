## The acceptance gate

```bash
cd ontology/cli-tools
python run_all_layers.py            # the gate
python run_all_layers.py -v         # per-code breakdown
python run_all_layers.py --update-golden   # accept current counts as the new reference
```

### Why reference values, and not pass/fail

**M1 has never been conformant.** A binary `CONFORMS` gate would have been red every day
for six months ‚Äî so it would have been ignored, so it would have been useless. That is
precisely how the debt grew *while the checkers existed*.

So the gate borrows the **TriskeleToolchain septuple-validation discipline**: exact
reference values.

> **163 is GREEN. 162 is RED. 164 is RED.**

| Move | Meaning |
|---|---|
| **164+** | **regression** ‚Äî a new defect was introduced |
| **162‚àí** | **also red.** Either you repaired something on purpose (then run `--update-golden`, and the drop is *recorded*), or **a validator stopped biting** ‚Äî a shape lost its target, a schema wasn't found, a file was skipped. |

**A silently shrinking error count is the most dangerous signal in this repo, because it
looks like progress.** The count is a **thermometer of the debt**, not a traffic light. It
is *allowed* to be non-zero. It is *not allowed* to move by accident.

## `tscg_paths.py` ‚Äî why it exists

The scripts used to open with:

```python
REPO_ROOT = Path("E:/_00_Michel/_00_Lab/_00_GitHub/tscg")
```

They never used their own location to find the repository. They worked on exactly one
machine, one OS, one clone ‚Äî and moving them anywhere was a no-op, because they still
pointed at `E:/`.

`tscg_paths` resolves the root by walking **up** from the script until it finds a directory
holding **both** `ontology/` and `instances/`. The scripts are therefore **relocatable**:
they run from `ontology/cli-tools/check-M1/`, from the legacy `cli_tools/check-M1/`, from a
CI box or a second clone, unchanged.

```python
# every script starts with this bootstrap (tscg_paths lives one level up)
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))

from tscg_paths import REPO_ROOT, ONTOLOGY_DIR, INSTANCES_ROOT, find_schema, verify_layout
```

Override when the layout is non-standard:

```bash
TSCG_REPO_ROOT=/path/to/tscg python check_M1.py
```

It is also the single source of truth for `INSTANCE_DIRS` and the canonical base URIs ‚Äî so a
compartment can no longer be spelled two different ways in two different scripts.

## ‚öÝÔ∏è The bugs these tools used to have ‚Äî all of the *silent* kind

SC-1 (2026-07-12) found four. None of them **crashed loudly** or **reported a failure**. They
either did nothing, or skipped things, while looking exactly like success.

| # | Bug | What it actually did |
|---|---|---|
| 1 | `check_M1.run()` called `fix_imports_genesis()` ‚Äî **a method that was never defined** | `AttributeError` on the first file loaded, in the auto-fix path. **None of the seven documented auto-fixes had ever run.** The tool meant to repair M1 was itself broken. |
| 2 | `check_M1` looked for `M1_Schema.shacl.ttl` (**dot**); the file is `M1_Schema_shacl.ttl` (**underscore**) | `--shacl` never found its grammar, **validated nothing, and exited 0**. |
| 3 | The M0 scripts looked for `symbolic-system-grammar` (**singular**); the directory is `symbolic-system-grammars` (**plural**) ‚Äî twice, in two different tables | `if not base_dir.exists(): continue` ‚Üí the **entire SymbolicSystemGrammar category** (Iching, TriskeleToolchain) was **never checked, never migrated**, without a word. |
| 4 | Hardcoded `REPO_ROOT` (see above) | One machine, one OS, one clone. |

> **A validator that cannot find its grammar looks exactly like a validator that passed.**
> This is why M1 kept accruing debt *while* the checkers "existed".

Mitigation, beyond the fixes: **`verify_layout()`** now *reports* every missing instance
compartment instead of swallowing it. A totals line computed over a silently skipped
category is worse than no totals line at all.

## What the M1 checker enforces since SC-1

A combo's formula **is** the signature of the function that produces it.

```
Fm2   : GenericConcept¬≤‚Å∫            ‚Üí  m2:GenericConceptCombo   (‚â• 2 named concepts)
Fm1m2 : Domain‚Å∫ , GenericConcept‚Å∫   ‚Üí  m2:DomainConceptCombo    (‚â• 1 domain AND ‚â• 1 concept)
```

`Fm2`/`Fm1m2` are **functions, not functors**: emergence is non-compositional (the arguments
are *combined, not associated*), and a functor must preserve composition. So a combo has **no
monoidal formula and no monoidal expansion**. `√ó`/`+`/`|` belong to **atoms only**.

| Code | Checks |
|---|---|
| `DCC000` | retired class `m2:KnowledgeFieldConceptCombo` ‚Üí `m2:DomainConceptCombo` (auto-fix) |
| `xxx006` | **monoidal operator inside a signature** ‚Äî the bulk of the SC-6 backlog (109 entries) |
| `xxx007` | **primitive type as an argument** (`A`, `St`, `F`, `It`, ‚Ä¶) ‚Äî a primitive is a generative *dimension*, not a concept |
| `xxx008` | wrong arity |
| `xxx009` | **guard appended after the signature** (`\| gain > 1`) |
| `DCC010` | **`Fm1m2`'s first argument is not a registered Domain** ‚Äî checked against `M1_Domains.jsonld` |
| `EXP001` | retired `m1:structuralGrammarFormulaExpanded` (D8 ‚Äî there is nothing to expand) |

**`DCC010` deserves a note.** A formula's arguments live only as **text inside a string** ‚Äî
never as IRIs (`m2:hasComboComponent` is absent from all of M1; decision **D10**). So no
purely syntactic check can tell a Domain from a Concept, and a domain-less `Fm1m2` such as

```
Fm1m2(Cascade, Duplication, Network)
```

passes every string test: right prefix, ‚â• 2 arguments, no operator. Neither regex nor SHACL
catches it. Loading the **domain registry** does. It is the closest we can get to enforcing
the grammar without reifying the arguments ‚Äî and it is what finally caught the defect whose
root was a wrong `Fm1m2` definition in `M3_GrammarFoundation` (*"crosses the M1/M2 boundary"*,
fixed in v2.4.0).

**Guards are not one thing.** `| gain_per_stage > 1` and `| Œª > 0` are **scalars** ‚Äî an M0
measurement that leaked a layer; M1 describes structure, not values. **Delete them.** But
`| trajectoryShape=Circular` is a controlled enumerated value carrying real semantics (the
*differentia specifica*). **Do not delete it** ‚Äî pending decision **D11**, ruled in SC-3
(an "option" with an orthogonal axis and a controlled value-set *is* a facet).

Full reference: `check-M1/check_M1_README.md` ¬∑
Rationale: `ontology/StructuralGrammar/Functional_Grammar_Model.md`
