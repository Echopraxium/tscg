## 📖 Définitions Détaillées

### O - Observability (Observabilité)

**Définition:**  
Degré auquel un système ou modèle peut être observé et ses états internes mesurés.

**Échelle:** [0, 1]
- **0.0** = Totalement inobservable (concept abstrait pur)
- **0.5** = Partiellement observable (système social)
- **1.0** = Complètement observable (expérience contrôlée)

**Exemples:**
- **High O (0.9):** Expérience de physique avec instruments de mesure précis
- **Medium O (0.5):** Étude psychologique avec auto-rapports
- **Low O (0.1):** Concept philosophique abstrait (ex: "beauté")

**Formule:** 
```
Observability(M) = |{observable_states}| / |{total_states}|
```

---

### R - Reproducibility (Reproductibilité)

**Définition:**  
Degré auquel les résultats peuvent être reproduits de manière cohérente dans des conditions similaires.

**Échelle:** [0, 1]
- **0.0** = Non reproductible (événement unique)
- **0.5** = Reproductible avec variance (sciences sociales)
- **1.0** = Parfaitement reproductible (mathématiques)

**Exemples:**
- **High R (0.95):** Expérience de chimie avec protocole strict
- **Medium R (0.6):** Étude clinique avec variation individuelle
- **Low R (0.1):** Événement historique unique

**Formule:**
```
Reproducibility(M) = 1 - variance(repeated_measurements)
```

---

### I - Interoperability (Interopérabilité)

**Définition:**  
Degré auquel le système s'intègre et interface avec d'autres systèmes.

**Échelle:** [0, 1]
- **0.0** = Système fermé, propriétaire
- **0.5** = Intégration partielle, APIs limitées
- **1.0** = Standards ouverts, interopérabilité totale

**Exemples:**
- **High I (0.9):** API REST avec standards OpenAPI
- **Medium I (0.5):** Système avec export CSV/JSON
- **Low I (0.1):** Logiciel propriétaire fermé

**Formule:**
```
Interoperability(M) = |{compatible_interfaces}| / |{total_interfaces}|
```

---

### V - Verifiability (Vérifiabilité)

**Définition:**  
Degré auquel les affirmations sur le système peuvent être vérifiées et validées.

**Échelle:** [0, 1]
- **0.0** = Non vérifiable (affirmations subjectives)
- **0.5** = Vérifiable empiriquement avec incertitude
- **1.0** = Vérifiable formellement (preuve mathématique)

**Exemples:**
- **High V (0.95):** Théorème mathématique avec preuve formelle
- **Medium V (0.6):** Étude empirique avec statistiques
- **Low V (0.2):** Témoignage personnel subjectif

**Formule:**
```
Verifiability(M) = |{verifiable_claims}| / |{total_claims}|
```

---

### E - Evolvability (Évolvabilité)

**Définition:**  
Degré auquel le système peut évoluer, s'adapter et être étendu au fil du temps.

**Échelle:** [0, 1]
- **0.0** = Système rigide, non modifiable
- **0.5** = Extension possible avec effort
- **1.0** = Architecture modulaire hautement extensible

**Exemples:**
- **High E (0.9):** Architecture microservices avec plugins
- **Medium E (0.5):** Code legacy avec points d'extension limités
- **Low E (0.1):** Système hardcodé monolithique

**Formule:**
```
Evolvability(M) = modularity × extensibility × maintainability
```
