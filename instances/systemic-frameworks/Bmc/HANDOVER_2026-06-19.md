### Tableau des namespaces TSCG (v1.5 définitif)

| Alias | Résout vers | Usage |
|---|---|---|
| `m3:` | `M3_GenesisGrammar.jsonld#` | Types M3, ontologyType |
| `m2:` | `M2_GenericConcepts.jsonld#` | GenericConcepts, formules |
| `m1:` | `M1_CoreConcepts.jsonld#` | CoreConcepts, domain, simulationTitle |
| `m1.extensions.<domain>:` | `M1_extensions/<d>/M1_<D>.jsonld#` | Concepts de domaine |
| `m0:` | `M0_Common.jsonld#` | Scores, gap, focal, enum **PARTAGÉS** |
| `m0.<instance>:` | `instances/.../M0_<Name>.jsonld#` | Propriétés **LOCALES** à l'instance |

**INTERDITS :**
- `m1core:` → utiliser `m1:`
- `m1ext:`, `m1bio:`, `m1chem:` → utiliser `m1.extensions.<domain>:`
- `m0bmc:`, `m0vsm:` → utiliser `m0.bmc:`, `m0.vsm:`
- `tscg:` → interdit
- `M0_Poclet#` → remplacé par `M0_Common.jsonld#`
- `A_score`, `S_score`... aliases → utiliser `m0:scoreA`, `m0:scoreS`...
- `{"@value": "0.85", "@type": "xsd:float"}` → utiliser `0.85` (bare numeric)
- `"Coherent"` pour spectralClass → utiliser `{"@id": "m0:spectralClass.Coherent"}`

## 6. Points conceptuels à retenir

**Score de Focalisation Épistémique (SFE)** = métaphore ophtalmologique nouvelle :
- δ₁ mesure l'écart Map/Territory (strabisme)
- SFE mesure la netteté de la fusion binoculaire (acuité visuelle)
- Orthogonaux : un système peut être bien aligné (δ₁ faible) mais astigmate (Gs absent)

**M0_Common.jsonld** est la réponse à la dette technique `M0_Poclet#` :
- Toutes les instances redéfinissaient les mêmes 10 aliases `A_score`... → 320 définitions redondantes
- Maintenant : 1 définition canonique + 33 imports

**La règle des valeurs bare** :
- `rdfs:range xsd:float` dans M0_Common.jsonld → `"m0:scoreA": 0.95` suffit
- `owl:ObjectProperty` + `owl:oneOf` → `{"@id": "m0:spectralClass.Coherent"}` obligatoire
